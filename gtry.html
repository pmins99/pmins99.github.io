Project 01

오픈소스 선제적 취약점 탐지 및 보안 대응

SAST Vulnerability Research 2025.12 — 2026.02 [토스뱅크] 사이버보안 부트캠프

오픈소스 소프트웨어는 현대 IT 인프라의 핵심이지만 기존 보안 대응은 취약점 발견 후 패치를 기다리는 수동적인 방식에 머물러 있습니다. 패치 배포까지의 공백 동안 취약점에 무방비로 노출되는 구조적 한계가 있습니다. 특히 동일한 오픈소스를 사용하는 다수의 서비스가 단일 취약점으로 동시에 위협받는 소프트웨어 공급망 문제로 이어질 수 있습니다. 이 프로젝트는 그 공백을 없애는 것을 목표로 했습니다. SAST 기반 정적 분석 체계를 구축하고 소스코드 분석을 통해 취약점을 선제적으로 식별했습니다. 단순 탐지에 그치지 않고 PoC 작성으로 실제 exploit 가능성을 검증하고 코드 레벨 패치 방안을 마련한 뒤 각 Security 팀에 공식 제보하여 CVE 2건을 발급받았습니다.

문제 정의

기존 방식의 한계

오픈소스 취약점 대응은 CVE 등록 이후 패치 적용에 의존한다. CVE 등록까지 평균 1개월, 패치 배포까지 추가 수 주가 소요되어 그 공백 동안 취약점에 무방비 상태로 노출된다.

선제적 대응 목표

패치 공개 이전 소스코드 분석을 통해 취약점을 먼저 식별한다. Exploit 가능성 기반의 우선순위 대응으로 제한된 자원을 실제 위협에 집중시킨다.

핵심 기술 · 담당 역할

취약점 분석 · PoC 작성

SAST 탐지 결과 중 실제 Exploit 가능성이 있는 취약점을 선별하고, PoC를 직접 작성하여 재현 가능성을 검증했다. 다양한 유형을 분석했으며, 코드 레벨 패치 방안까지 제시했다.

Custom Rule 설계

Default Ruleset이 탐지하지 못하는 패턴을 분석하여 Semgrep Custom Rule을 직접 설계했다. Path Traversal, postMessage Origin 미검증 등 일반화된 패턴으로 포착하기 어려운 취약점에 특화된 규칙을 작성했다.

LLM 기반 자동화 파이프라인

SAST 결과의 대량 FP를 효율적으로 처리하기 위해 LLM 기반 자동화 파이프라인을 구축했다. Pre-scan 필터링, SARIF 정규화, CoVe·RAG 기반 LLM 검증을 단계적으로 적용해 노이즈를 54.7% 감소시켰다.
